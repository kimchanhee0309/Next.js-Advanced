<details>
  <summary>05-1. Router Cache</summary>

  * 클라이언트 내에 RSC Payload(화면렌더링에 필요한 정보)를 저장
  * Link 컴포넌트의 prefetch로 RSC Payload를 저장하는 공간
  * 페이지 방문 시 Cache SET, 페이지 재방문 시 CacheHIT
  * 브라우저 세션동안만 유지되는 캐시(브라우저창 닫거나, 새로고침 시 초기화)
  * **RSC Payload란?**
    * React 컴포넌트 트리의 직렬화된 표현으로, 클라이언트에서 하이드레이션에 사용됨
</details>

<details>
  <summary>05-2. Full Router Cache</summary>

  * 서버에서 프리렌더링 결과인 html, RSC Payload 저장되는 공간
  * npm run build 했을 때 저장되는 공간
  * Full Route Cache `SET`되는 경우
  * SSC / ISR / generateStaticParams
</details>

<details>
  <summary>05-3. Request Memoization</summary>

  * 같은 렌더링 과정에서 발생하는 fetch 요청을 일시적으로 캐싱하는 공간
  * 단일 렌더링 과정이 끝나면 자동으로 캐시에서 제거됨
  * 예시 코드
    ```javascript
    // 같은 렌더링 사이클 내에서 동일한 fetch 호출
    async function Page() {
      // 첫 번째 fetch - 실제 네트워크 요청 발생
      const data1 = await fetch('https://api.example.com/data', { cache: 'force-cache' });

      // 두 번째 동일한 fetch - request memoization HIT!
      const data2 = await fetch('https://api.example.com/data', { cache: 'force-cache' });

      // 두 변수는 동일한 데이터를 참조
      const result1 = await data1.json();
      const result2 = await data2.json();

      // ...
    }
    ```

    * fetch api를 사용하지 않더라도 Request Memoization 사용하는 방법
      * react의 cache를 사용하면 됨
      ```javascript
      import { cache } from 'react'

      // 사용자 정의 함수 메모이제이션
      export const getUser = cache(async (id) => {
        // 비용이 많이 드는 연산 또는 데이터 처리
        const user = awwait db.query(...);
        return transformUserDate(user);
      })

      // ----- 특정 페이지 컴포넌트에서 getUser 반복호출 케이스
      export default async function Page() {
        const a = await getUser(1);  // DB 호출 #1(id:1) - 실제 실행
        const b = await getUser(1);  // 캐시 HIT  - 본문 실행 안 됨
        const c = await getUser(1);  // 캐시 HIT
        const d = await getUser(2);  // DB 호출 #2(id:2) - 인자가 다르므로 새 항목

        // 콘솔에는 "DB 호출 #1", "DB 호출 #2" 두 줄만 찍힘
        // queryCount === 2
      }
      ```

<details>
  <summary>05-5. fetch 캐시 옵션 정리</summary>

  ```javascript
  fetch(url);
  fetch(url, { cache: "force-cache" });
  fetch(url, { cache: "no-store" });
  fetch(url, { next: { revalidate: 5 } });
  ```

  |캐싱 옵션|식품 비유|주요 특징|적합한 데이터|
  |---|---|---|---|
  |`force-cache`|냉장고에 있는 식품 그대로 사용|캐시 있으면 무조건 사용, 서버 확인 안 함|정적 콘텐츠, 변경 드문 데이터|
  |`no-store`|냉장고 사용 안 함, 항상 새로 구매|캐시 사용 안 함, 항상 서버에서 가져옴|실시간 데이터, 개인화된 정보|
  |`revalidate:N`|타이머 달린 냉장고 식품|설정 시간 내 캐시 사용, 이후 확인|주기적으로 변경되는 데이터|
</details>

<details>
  <summary>05-6. 캐시 무효화 방법</summary>

  #### `revalidatePath()`
  * 서버 환경에서 특정 경로의 모든 캐시 레이어 무효화(새로고침과 유사한 효과)
  * **무효화되는 캐시**
    * Router Cache(클라이언트)
    * Full Route Cache(서버)
    * Data Cache(서버)
  * **사용 위치**: 서버 컴포넌트, 서버 액션, 라우트 핸들러
  ```javascript
  // 예시
  'use server';
  import { revalidatePath } from 'next/cache';

  async function updatePost(formData) {
    await saveToDatabase(formData);

    // /blog와 관련된 모든 캐시를 무효화
    revalidatePath('/blog/[id]', "page");
  }
  ```

<br>

  #### `revalidateTag()`
  * 서버 환경에서 특정 태그가 붙은 데이터 캐시만 무효화(비교적 부드러운 리렌더링 효과)
  * **무효화되는 캐시**
    * Data Cache(태그 관련)
    * 간접적으로 관련 Route Cache(데이터 변경 시)
    * 관련 라우트의 Full Route Cache
  * **사용 위치**: 서버 컴포넌트, 서버 액션, 라우트 핸들러
  * **주요 특징**:
    * 더 세밀한 제어 가능
    * 특정 데이터 세트만 타겟팅
    * 불필요한 캐시 무효화 방지
  ```javascrpit
  // 데이터 fetch 시 태그 지정
  async function getProducts() {
    const res = await fetch('https://api.example.com/products', {
      next: { tage: ['products'] }
    });
    return res.json();
  }

  // 태그로 무효화
  'use server';
  import { revalidateTag } from 'next/cache';

  async function updateProduct(id, data) {
    await updateInDatabase(id, data);

    // 'products' 태그가 있는 모든 데이터 캐시를 무효화
    revalidateTag('products');
  }
  ```

<br>

  #### `router.refresh()`
  * 클라이언트 환경에서 현재 경로 새로고침(서버 컴포넌트 재실행)
  * **영향 범위**: 현재 경로의 Router Cache만
  * **무효화되는 캐시**:
    * O, Router Cache(현재 경로만)
    * X, Full Route Cache
    * X, Data Cache
    * X, Request Memoization
  * **사용 위치**: 클라이언트 컴포넌트만
  * **주요 특징**:
    * 가장 범위가 좁음
    * 클라이언트에서만 호출 가능
    * 현재 페이지에만 영향
    * 서버 컴포넌트 재실행(새로운 데이터 요청)
  ```javascript
  'use client';
  import { useRouter } from 'next/navigation'

  export function RefreshButton() {
    const router = useRouter();

    return (
      <button onClick={() => router.refresh()}>
        현재 페이지 새로고침
      </button>
    );
  }
  ```
